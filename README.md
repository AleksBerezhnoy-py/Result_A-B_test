# Анализ результатов A/B-тестирования новой механики оплаты

## Описание проекта
В ходе тестирования продуктовой гипотезы целевой группе пользователей маркетплейса была предложена новая механика оплаты услуг на сайте, в то время как у контрольной группы осталась базовая механика. Цель проекта — проанализировать результаты эксперимента, оценить влияние изменений на ключевые метрики (CR, ARPU, ARPPU) и сформировать бизнес-рекомендацию: стоит ли раскатывать новую механику на всех пользователей.

Полный код исследования, статистические тесты и графики находятся в Jupyter Notebook: [`Project.ipynb`](Project.ipynb).

## Стек технологий и методов
* **Язык:** Python
* **Библиотеки:** Pandas, Scipy.stats, Seaborn, Matplotlib, Numpy
* **Статистические методы:** Критерий Хи-квадрат, Bootstrap (для оценки разницы средних в ненормальных распределениях).

## Входные данные
* `groups.csv` — информация о принадлежности пользователя к контрольной (А) или целевой (B) группе.
* `groups_add.csv` — дополнительный список пользователей, полученный через 2 дня после начала теста.
* `active_studs.csv` — список пользователей, заходивших на платформу в дни эксперимента.
* `checks.csv` — данные о совершенных оплатах.

## Результаты:

1. **Конверсия в оплату (CR):** Анализ таблиц сопряженности с помощью критерия Хи-квадрат показал `p-value = 1.0` (различий нет). Новая механика оплаты не повлияла на общую долю платящих пользователей от общего числа зашедших.
2. **Выручка на пользователя (ARPU):** Расчет разницы средних доходов методом Бутстрапа показал, что 95% доверительный интервал равен `[-0.19, 3.61]` и включает в себя ноль. Изменения в ARPU статистически не значимы.
3. **Средний чек платящих (ARPPU):** Бутстрап-оценка разницы средних чеков платящих пользователей показала 95% доверительный интервал `[71.78, 407.73]`. Интервал не содержит ноль, что доказывает статистически значимый рост среднего чека в целевой группе (B).

## Бизнес-рекомендация
Несмотря на то, что новая механика оплаты не привлекла новых покупателей, она успешно монетизирует текущую базу, стимулируя платящую аудиторию совершать более крупные покупки. Фича экономически эффективна, рекомендуется раскатывать изменения на 100% пользователей.

## Инструменты автоматизации
В ноутбук добавлены готовые функции:
* `update_metrics` — для автоматического объединения основных/дополнительных логов и мгновенного пересчета CR, ARPU и ARPPU.
* `plot_metrics` — для автоматического построения сравнительных графиков (Barplot, Boxplot, Violinplot) по группам.
